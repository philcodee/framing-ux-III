### Saved form state (Supabase `saved_forms` table)
Implemented full save/restore flow for in-progress forms.

**New functions:**
- `saveFormState(stage)` — upserts to `saved_forms`; inserts on first call and stores the row id, updates on subsequent calls. Saves fields, values, fillGroupIndex, scanSummary.
- `deleteSavedForm()` — deletes the row; called on Start Over.
- `checkSavedForm(userId)` — queries for the most recent saved row on boot.
- `restoreFormState(saved)` — repopulates state and jumps to Fill (phase 3) or Review (phase 4) depending on the saved stage.
- `showRestoreOffer(saved)` — shows the restore screen with the doc type in the prompt.
- `proceedAfterAuth()` — extracted boot continuation logic, shared between normal boot and the "Start fresh" path.

**Save trigger points:**
- After scan confirms → `saveFormState('fill')`
- After each fill group Next → `saveFormState('fill')`
- Before advancing to Review → `saveFormState('review')`
- On Start Over → `deleteSavedForm()`

**Restore screen:**
New `#restore-screen` overlay (Carbon-styled, matching profile/auth card pattern). Shows doc type in the prompt. Two actions: "Continue my form" (restores state, jumps to correct phase) and "Start fresh" (deletes saved row, proceeds to normal boot).

**Boot flow change:**
After auth, `bootApp` now checks for a saved form before showing the profile screen. If found, restore screen is shown instead.

**Note:** The original uploaded file is not restored (not persisted — too large for JSONB). The extracted field data and all user answers are, so the user picks up at the exact fill group they left off without needing to re-upload or re-scan.

### Carbon consistency audit
Generated `CARBON_CONSISTENCY_REPORT.md` — full audit of all components against Carbon v11 spec.

**4 high-priority findings:**
1. `.ug-back-btn` is a hybrid not matching any Carbon button variant (secondary, tertiary, or ghost)
2. Three locations use raw `rgba(244,244,244,...)` values instead of named tokens
3. Dark theme missing `--cds-support-success: #42be65` (G90 value — contrast issue on dark bg)
4. `.fill-sensitive` tag missing `background: var(--cds-support-error-bg)`

**5 medium findings:** missing `min-width` on primary button, inline `style` attributes with off-scale spacing (10px), missing `text-decoration` on review edit link, hardcoded `160px` column width, missing `line-height` on body element.

Custom components (upload zone, stage track, signature canvas, agent brief) flagged as intentional — no Carbon equivalent, but all use tokens correctly internally.

Fixes not applied this session — deferred to a future pass.
