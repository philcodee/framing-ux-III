### App restructure — v2
Deprecated the old 6-phase wizard (`civicguide-v1.html`). Rebuilt from scratch as a document-first 5-phase flow:

| Phase | What happens |
|-------|-------------|
| 1 — Upload | File drop/select, PDF or image, max 4MB |
| 2 — Scan | Gemini reads document, returns structured JSON, user confirms domain + doc type |
| 3 — Fill | 3 fields at a time, AI pre-filled, profile-aware, signature canvas |
| 4 — Review | All fields in a table, edit buttons jump back to Fill |
| 5 — Export | Confirmation screen |

### Key decisions
- **Location** collected in profile form, not in wizard
- **Domain** inferred from uploaded document by AI — no selection phase needed
- **All AI activity confirmed by human** before advancing
- **Profile pre-fill** — Gemini receives name + location from profile and uses them to pre-populate matching fields
- **Signature pad** — canvas-based, mouse and touch, stored as data URL
- **Field pagination** — 3 fields per page with progress pips to reduce overwhelm

### Gemini notes
- Active model: `gemini-2.5-pro` (switched from `gemini-2.5-flash` which was consistently 503)
- `gemini-2.0-flash` and `gemini-1.5-flash` return 404 for this API key (only 2.5 models available to new users)
- Retry logic: 4 attempts at 0 / 2s / 4s / 8s for 503 errors
- Max output tokens for scan: 16,384 (needed for large forms with many fields)
